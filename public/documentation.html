Reel Creations API

This web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Endpoints

Business Logic URL HTTP Method Request body data format Response body data format
Return a full list of all movies /movies GET None JSON object of all movies and their keys
Add a new Movie to the database /Movies POST JSON object with keys: title, description, genre, director, image_path, actors(array)
example: {"title" : "Schindler's List", "description" : "1993 World War II Movie about Schindler", "genre" : "5d1a54e098397de8da1fc574", "director" : "5d1a56dd98397de8da1fc577", "image_path" : "SchindlersList1993.jpg", "featured" : true, "actors" : ["Liam Neeson", "Ben Kingsley"] } 		confirmation message: reiterating info submitted
Return information on a single movie /movies/[title] GET None JSON object with keys: title, description, genre, director, imageURL, featured(bool)
Return a full list of genres /genres GET JSON objects of all genres and their keys
Return information on a single genre /genres/[name] GET JSON object with keys: name, description
Return a full list of directors /directors GET None JSON object of all directors and their keys
Return information on a single director /directors/[name] GET None JSON object with keys: name, bio, birthYear, deathYear
Register a new user /users POST JSON object with keys: id, username, password, email, dob, favorites(array)
example: { "username":"jpjones", "name":"John Paul Jones", "password":"thereandbackagain123", "email":"jpjones@notmadeup.com", "birth_date":"1991-02-14", "favorites":[ "Wonder%20Woman", "Saving%20Private%30Ryan" ] } 		confirmation message: reiterating info submitted
Update a user's information /users PUT JSON object with keys: id, username, password, email, dob, favorites(array) confirmation message: reiterating info submitted
Add a movie to a user's favorite list /users/[id]/[title] POST None confirmation message: reiterating info submitted
Remove a movie from a user's favorite list /users/[id]/[title] DELETE None confirmation message: reiterating info submitted
Unregister a user /users/[id] DELETE None confirmation message: reiterating info submitted
